Nuevas etiquetas de html5

A continuación se muestra una serie de etiquetas que han sido introducidas en HTML5:

Etiquetas de cabecera

  • <header>
    • Esta etiqueta recoge todo el contenido de la cabecera de la página, como el menú de navegación, las imagenes principales, el botón de inicio, entre otros botones.
  • <nav>
    • Esta etiqueta define la sección de navegación, como menú de acceso rápido a diferentes apartados de la página:
    • Menú 1
    • Menú 2
    • Menú 3
    • Menú 4
    • Menú 5

Etiquetas de cuerpo principales

  • <main>
    • Esta etiqueta muestra el contenido principal de una página, esta puede aparecer una unica vez en el documento.
  • <article>
    • Esta etiqueta representa un fragmento independiente de contenido, esto puede ser un artículo o una entrada de un blog.
  • <section>
    • Esta etiqueta permite agrupar contenido por tipo, también nos permite mostrar pequeños fragmentos de información.
  • <aside>
    • Esta etiqueta muestra una parte de información ligeramente relacionada con el resto de la página, esta suele estar de forma reducida en el lateral.

Etiquetas multimedia

  • <canvas>
    • Nos permite dibujar gráficos por pantalla.
  • <video>
    • Esta etiqueta nos permite insertar un video de forma externa o interna.

  • <audio>
    • Como en la etiqueta anterior, pero en este caso con un audio.
    • Ejemplo de forma interna:
  • <embed>
  • Esta etiqueta nos permite insertar contenido externo dentro de un archivo HTML, este puede ser PDF, videos o diferentes fuentes multimedia, entre otros.
  • <source>
    • Especifica recursos de medios múltiples para elementos <picture>, <audio> o <video>.
  • <track>
    • Este elemento se utiliza como elemento hijo de los elementos multimedia <audio> y <video>.

Etiquetas de formato

  • <wbr>
    • Esta etiqueta permite insertar un salto de línea cuando un texto sea demasiado largo y no se ajuste a un contenedor o a la misma pantalla.
  • <mark>
    • Esta etiqueta permite resaltar una palabra o un texto.
  • <figure>
    • Esta etiqueta nos permite ilustrar artículos, fotos, diagramas, ilustraciones, entre otros.
  • <figcaption>
    • El pie o explicación de un elemento figure
  • <bdi>
    • Esta etiqueta da una definición extra.
  • <menu>
    • Lista de opciones pertenecientes a un menú.
  • <command>
    • Elemento de un menú de opciones.
  • <details>
    • Detalles o información suplementaria que se puede ocultar o ver.
  • <summary>
    • Encabezado para detalles especificados en el elemento details
  • <hgroup>
    • Grupo de encabezamientos, útil para cuando hay diversos bloques de encabezados del mismo nivel H1, H2...
  • <time>
    • Elemento para escribir una fecha, una hora o ambas.
  • <dialog>
    • Este elemento nos permite representar conversaciones.

Etiquetas de formulario

  • <form>
    • Esta etiqueta define un formulario HTML

      • Atrbutos:

      1. accept
        • Lista separada por comas de los tipos de contenido que el servidor acepta.
      2. accept-charset
        • Lista de codificación de caracteres que el servidor acepta.
      3. action
        • La URI que procesa la información enviada por el fomulario.
      4. autocomplete
        • Indica si se pueden rellenar automáticamente valores de forma predeterminada.
      5. enctype
        • Cuando el valor del atributo method es post, este atributo es el tipo MIME del contenido que es usado para enviar el formulario al servidor.
      6. method
        • El método HTTP que utiliza el navegador para enviar el formulario.
      7. name
        • El nombre del formulario.
      8. novalidate
        • Este atributo de tipo booleano indica que el formulario no será validado en el envio.
      9. target
        • Un keyword indicando donde debe mostrarse la respuesta que es recibida después de enviarse el formulario.
    • <input>
      • Esta etiqueta nos permite crear controles interactivos para el formulario que nos permite recibir los datos del usuario.

      Atributos:

      • Type
        • Indica el tipo de campo, cada uno tiene unas características.
          1. button
            • Botón sin un comportamiento especifico.
          2. checkbox
            • Casilla de selección, se debe utilizar el atributo value para definir el valor que se enviará por este.
          3. date
            • Control para introducir una fecha (año, mes y dia, sin tiempo).
          4. email
            • Campo para introducir una dirección de correo electrónico.
          5. file
            • Campo que permite al usuario seleccionar un archivo.
          6. hidden
            • Campo no mostrado al usuario pero cuyo valor es enviado al servidor.
          7. number
            • Campo para introducir un número de punto flotante.
          8. password
            • Campo que permanece oculto.
          9. radio
            • Se debe usar el atributo value para definir el valor que se envia con el elemento.
          10. tel
            • Campo para introducir un número de telefono, este elimina automáticamente los salto de línea.
          11. text
            • Campo de texto de línea simple.
          12. reset
            • Botón que restaura los contenido de un formulario a sus valores predeterminados.
          13. submit
            • Botón que envia el formulario.
      • autofocus
        • Este atributo booleano permite especificar que un control del formulario tenga el cursor cuando la página se cargue.
      • checked
        • Si el valor del atributo type es radio o checkbox la presencia de esta atributo booleano indica que esta seleccionado.
      • disabled
        • Este atributo booleano indica que el control no esta listao para la interacción.
      • max
        • El valor máximo para este elemento (numérico o fecha-hora).
      • maxlength
        • Si el valor del atribto type es text, email, search, password, tel o url, este atributo especifica el número máximo de caracteres que el usuario puede introducir.
      • name
        • El nombre del campo, este es enviado con los datos del formulario.
      • pattern
        • Una expresión regular con la que el valor se esta evaluando.
      • placeholder
        • Información dentro del campo para el usuario.
      • readonly
        • Este atributo indica que el usuario no puede modificar el valor del campo.
      • required
        • Este atributo le especifica al usuario que el campo debe de rellenarse obligatoriamente.
      • size
        • El tamaño inicial del campo.
  • <meter>
    • Esta etiqueta nos permite trabajar con medidas y escalas.
  • <progress>
    • Esta etiqueta implementa barras de progreso.
  • <datalist>
    • Extensión para crear campos con funcionalidad de autocompletar.
  • <keygen>
    • Genera claves pública y priada para encriptación.
  • <output>
    • Realiza y muestra cálculos matemáticos.
